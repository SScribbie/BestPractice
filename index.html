About

TiLT is a tool for locking text in one or more xliff files by using one or more search expressions. The search expressions could be either plain text or regular expressions (pattern match objects).

You add the search expressions to a list and decide if the found text should be classed as External or Internal. Then add one or more xliff files and execute.

Example with xml file where TiLT is useful:

<?xml version="1.0" encoding="utf-8"?>
<items>
	<item>CODE OF BUSINESS CONDUCT</item>
	<item>Welcome %1$s! Please view your signing status in the below overview.</item>
	<item>Once you have reviewed the Code of Conduct, click the "Acknowledge" button.</item>
	<item>By proceeding, you also agree to the &lt;font color="%1$s"&gt;Terms of Service
	&lt;/font&gt; and &lt;font color="%2$s"&gt;Privacy Policy&lt;/font&gt;.</item>
</items>

First we create an xliff from this xml by using Translation Workspace Convert Tool. The output will look like this in Translation Workspace Xliff Editor:

Now we want to lock code like %1$s and %2$s. We also want to lock the markup code <font ...></font>. Neither should the "Acknowledge" button name be translated. The client has also decided that the text Code of Conduct and CODE OF BUSINESS CONDUCT should be left untouched.

Working with patterns

Buttons
Add pattern If there are no characters in the pattern editor text field the cursor will be put in the pattern editor text field so you can start typing a pattern.
If there are any characters in the pattern editor text field the text will be added to the pattern list.
Edit selected pattern Edit the selected pattern. You can also do this by double-clicking a pattern in the list.
Remove selected pattern Remove the selected pattern from the list.
Remove all patterns Remove all patterns from the list.
NOTE! TiLT handles pattern files about the same as any text editor. This means that if you have a pattern file open (see Open patterns file) and remove all patterns from the list, the file is still open. So if you now save the file you lose all previous data you have saved in that file.

 

Add patterns

Patterns are added by creating them in the pattern editor in the bottom left in the TiLT main window. Enter a pattern in the pattern editor text field, add one or more flags by checking the check boxes.

If TiLT should recognize the pattern as a text string you select the <plain> option in the leftmost drop-down list. If you want TiLT to recognize the pattern as a regular expression select the <regex> option in the leftmost drop-down list. For help regarding regular expressions see Links.

You also have to decide whether the found text in the target xliff file(s) should be marked as External or Internal. You do this by selecting one of the two options in the rightmost drop-down list.

See also Preview for an alternative way to add patterns by selecting them from the opened xliff file.

Pattern tips!

Search for whole word only - Lets say you want to search for the word is in the sentence This text is a test text.
If you search for is as plain text you will get two matches: This text is a test text, which might not be what you wanted.
To solve this you can use the \b regex empty string operator. Searching the example text above with the regular expression \bis\b will now match the whole word only: This text is a test text.

Explanation from the official Python regex documentation: \b Matches the empty string, but only at the beginning or end of a word. A word is defined as a sequence of alphanumeric or underscore characters, so the end of a word is indicated by whitespace or a non-alphanumeric, non-underscore character. Note that formally, \b is defined as the boundary between a \w and a \W character (or vice versa), or between \w and the beginning/end of the string.

 

Edit patterns

If you want to edit a pattern you have added to your list you can do that by selecting it in the list and clicking the <Edit selected pattern> button (or use accelerator keys).

The pattern will be populated in the pattern editor for you to edit. When you are done editing you add the edited pattern to the list by clicking the <Add pattern> button (or use accelerator keys).

 

Remove patterns

If you want to remove a pattern you have added to your list you can do that by selecting it in the list and clicking the <Remove selected pattern> button (or use accelerator keys).

If you want to remove all patterns in the list you click the <Remove all patterns> button (or use accelerator keys).

NOTE! TiLT handles pattern files about the same as any text editor. This means that if you have a pattern file open (see Open patterns file) and remove all patterns from the list, the file is still open. So if you now save the file you lose all previous data you have saved in that file.

 

Disable/Enable patterns

If you want to keep a pattern but temporarily not want to use it you can do that by selecting it in the list and unchecking the enabled checkbox (or use accelerator keys).

You can enable a disabled pattern by selecting it in the list and checking the enabled checkbox (or use accelerator keys).

Unselected enabled patterns in the list have a black font color. Unselected disabled patterns in the list have a grey font color.

Options

Right above the pattern list you find the global options for the currently loaded patterns.

In the <Pre-process Filter> drop-down list you set the parsing filter for TiLT. By default TiLT only parse the translatable segments in an xliff but you have the option to let TiLT parse without filter ie. all segments will be affected by TiLT pre-process.

By checking the <Add equiv-text attribute> checkbox the equiv-text attribute will be added to all TiLT inline code. The value of the equiv-text attribute is the deserialized inline code. If an equiv-text attribute is present Translation Workspace Xliff Editor will display that value instead of the actual code. If you uncheck the <Add equiv-text attribute> checkbox the inline code will be displayed as is. The <Add equiv-text attribute> checkbox is checked by default.

The image above shows a segment with serialized code before TiLT pre-process.

The image above shows the TiLT pre-processed segment with the equiv-text attribute added.

The image above shows the TiLT pre-processed segment without the equiv-text attribute.

NOTE! Adding the equiv-text attribute to the xliff do not affect the code. It is stored as is. It is just a way to display the code nicely to the translator.

 

Save patterns to file

You save your patterns by clicking the <Save> or the <Save as...> button, or from the <File> menu. The patterns are stored in an xml format with the .tilt file extension. The options will be stored in the file as well.

Example of how it looks if you open it in a text editor (Notepad or similar):

<?xml version="1.0" encoding="utf-8"?>
<patterns>
	<options equiv-text="true" filter="translatable"/>
	<pattern enabled="true" type="plain" output="external" flags=""><![CDATA[CODE OF BUSINESS CONDUCT]]></pattern>
	<pattern enabled="true" type="plain" output="internal" flags=""><![CDATA["Acknowledge"]]></pattern>
	<pattern enabled="false" type="plain" output="internal" flags="I"><![CDATA[Code of Conduct]]></pattern>
	<pattern enabled="true" type="regex" output="internal" flags=""><![CDATA[<.+?>]]></pattern>
	<pattern enabled="true" type="regex" output="internal" flags=""><![CDATA[%[0-9]\$s]]></pattern>
</patterns>

Pre-processing

Buttons and checkboxes
Add Add xliff files for pre-processing. You add files by folder.
Remove Remove selected file(s) from the list.
Clear Remove all files from the list.
Go Run the pre-process.
Save Log Save the result to an html log file.
Dry run When checked TiLT do not save the files. This way you can test the patterns on the files and see the outcome.

Select the <Pre Process> tab and add one or more files to the file list. You do that by clicking the <Add> button or using drag-and-drop.

You can remove one or more files from the file list. Select one or more files in the list and click the <Remove> button, which will remove all selected files from the list. Remove all files from the list by clicking the <Clear> button.

When you are ready adding files and have the patterns you want to apply loaded, click the <Go> button to execute the pre-process.

Note! TiLT only processes source nodes. This means that you only use TiLT pre-processing on xliff files before they have been segmented/translated. If you pre-process a segmented/translated file TiLT will show a warning. The source node will have been pre-processed though.

Note! TiLT is developed for xliff files created by Translation Workspace Tools. If you use TiLT on third party xliff files it might not work properly. It is important that you check any third party xliff file thoroughly after pre-processing it. TiLT will warn when pre-processing third party xliff files.

Note! x-text: You will get a notification when pre-processing xliff files that has been created using Translation Workspace Text Filter. The Translation Workspace XML parser and the Translation Workspace Text File parser escape and unescape markup code differently. This could lead to markup escaping discrepancies in the post-processed file.
Make sure to double check markup code after post-processing xliff files created with the Translation Workspace Text filter.
Tip! Do to not use the Translation Workspace Text filter for preparing markup files (xml, html...).

The image above shows a finished pre-process run. Files with errors will be highlighted in red and files with warnings will be highlighted in yellow.

NOTE! Error files have not been pre-processed and need to be checked. The error could for example be an invalid xliff, problems with locating the path, the file might be write protected or you might not have write permissions in the folder where it resides.

Preview

Buttons and checkboxes
Add Add an xliff file for preview.
Clear Clear the preview window. This closes the loaded xliff file.
Update View Apply the patterns in the pattern list to the loaded xliff file.
Save Save the loaded xliff file.
Save As... Save the loaded xliff file as...
Autoupdate view When checked TiLT applies the patterns in the pattern list every time it changes i.e. when a pattern is added, edited or removed.
Show skeleton When checked, and if the xliff file has a skeleton (non-translatable data), the skeleton will be shown in the preview together with the segments.
Show tu and id When checked all segment id's will be shown in the preview together with the segments.

Preview is a key feature in TiLT. Here you can test patterns on an xliff file and instantly see the expected results. You can choose to save the loaded xliff file after applying the patterns.

Select the <Preview> tab and load an xliff file. You do that by clicking the <Add> button or using drag-and-drop.

The default view show the text in all segments separated by a line break, see image above.

Translatable text is colored black. Non-translatable text (i.e. segment is set to translate="no") is colored light grey with a grey highlight. External segments are colored light grey. Non-translatable Internal text is colored red.

If the xliff file has a skeleton file (non-translatable data outside the segments), the skeleton will be shown in the Preview together with the segments, see image above.

In this view segments are not necessarily separated by a line break. The skeleton data decides where line breaks occur.

You add the skeleton to the Preview by clicking the <Show skeleton> checkbox.

If you want to see the segment boundary and the segment id, you check the <Show tu and id> checkbox. The segment boundaries are shown as xml tags named with the id, see image above.

You can choose to have both skeleton view and tu view active at the same time, see image above.

You apply patterns to the Preview by adding them to the pattern list and then updating the Preview. You update the Preview by clicking the <Update view> button (or use accelerator keys), see image above.

If you check the <Autoupdate view> checkbox, TiLT automatically applies the patterns to the Preview as soon as any change occurs in the pattern list, i.e. when a pattern is added, edited or removed.

Another feature in the Preview is the possibility to add patterns from the text in the loaded xliff file.

Select a text, right click and select either <Add Selection as External> or <Add Selection as Internal> from the right click menu, see image above. The selected text will now be appended to the pattern list, see image below.

 

Change Preview Font

You can change font and font size for the Preview from the <Settings> menu or by right clicking the Preview and select <Set a Preview Font> from the right click menu.

 

Search in Preview

You can search the Preview using a loaded pattern as a search criteria.

Select a pattern in the patternlist, right click and select <Find Next in Preview> to search Down in the loaded Preview file or <Find Previous in Preview> to search Up in the loaded Preview file (or use accelerator keys). If a match is found it will be selected in the Preview, see image below.

 

View process details

By clicking this button after a TiLT pre-process, a detailed view will open (you can also use accelerator keys).

The detailed view will list the amount of changes made and markup type for each pattern from the previous pre-process execution.
You can store the results in an html log file by clicking the Save button.

Clicking the Hide detailed view / Show detailed view links in the top right corner of the log file toggles the detail level, see images below.
